# BusinessClaw

Персональный ИИ-ассистент для бизнеса на базе [OpenClaw](https://github.com/steipete/openclaw).  
Позволяет анализировать ключевые показатели продаж (выручка, конверсия, допродажи), делать прогнозы и получать уведомления о критических ситуациях.

---

## 🧠 Архитектура

<img width="1617" height="653" alt="image" src="https://github.com/user-attachments/assets/51ffdc1d-c9ed-402d-873b-345197add178" />

ИИ‑агент использует **Google Sheets** как простую и наглядную базу данных.  
Проверки критических показателей выполняются по расписанию через встроенный в OpenClaw **cron**.  
Нативный Telegram‑бот поддерживает кастомные команды (например, `/summary`, `/check`), а вся логика поведения агента описывается в файле `AGENTS.md`.  

Расчёты делегируются **генерируемым скриптам на Python**, что гарантирует точность и переиспользование.  
Ведётся подробное логирование: каждый запрос сохраняется в `memory/YYYY-MM-DD.md`, а результаты автоматически фиксируются в столбце `заметка` таблицы.

Конфигурация происходит через человеко‑читаемые файлы (`CONFIG.md`, `.env`), а установка возможна как поверх существующего деплоя OpenClaw, так и в отдельном Docker‑контейнере.

---

## 📊 Google Sheets

Данные хранятся в виде плоской таблицы, где каждая строка соответствует одному дню.  
Формат колонок:

| Колонка              | Тип     | Описание                                                                 |
|----------------------|---------|--------------------------------------------------------------------------|
| `дата`               | Дата    | ГГГГ-ММ-ДД. Пропущенные дни допустимы.                                  |
| `выручка`            | Число   | Общая выручка за день.                                                   |
| `количество лидов`   | Число   | Новые лиды (заявки).                                                     |
| `количество продаж`  | Число   | Закрытые продажи.                                                        |
| `конверсия`          | Число   | Необязательно – может вычисляться как продажи / лиды.                    |
| `сумма допродаж`     | Число   | Сумма дополнительных продаж (апсейлы).                                   |
| `средний чек`        | Число   | Выручка / продажи (если продажи > 0).                                    |
| `план продаж`        | Число   | Плановая выручка на день.                                                |
| `заметка`            | Текст   | Произвольные заметки (агент дописывает сюда свои выводы автоматически). |

Все вычисления (конверсия, средний чек) выполняются скриптами; в таблице они могут отсутствовать.  
Агент при необходимости создаёт недостающие колонки автоматически.

## 🔧 Установка

### 1. Клонирование репозитория и настройка окружения

```bash
git clone https://github.com/indiscrete-void/businessclaw.git
cd businessclaw
cp .env.example .env
```

В `.env` нужно будет указать идентификатор таблицы, email Google‑аккаунта и пароль для шифрованного файла авторизации.

### 2. OpenClaw

Выберите один из вариантов установки.

#### 2.1.A. Локальная установка OpenClaw

Следуйте [официальной инструкции](https://docs.openclaw.ai/install).  
Дополнительно установите утилиту `gog` для работы с Google Sheets:

```bash
brew install gogcli
# или
git clone https://github.com/steipete/gogcli.git
cd gogcli
make
cp ./bin/gog /usr/local/bin/gog
```

#### 2.1.B. Docker‑установка

Если вы предпочитаете Docker, воспользуйтесь [инструкцией](https://docs.openclaw.ai/install/docker).  
После этого скопируйте в директорию проекта файлы `Dockerfile.local` и `docker-compose.override.yml` – они содержат патчи для корректной работы `brew` и `npm`.  
`gog` установится автоматически.

#### 2.2. Настройка OpenClaw

Запустите стандартную процедуру настройки: подключите нужную модель, укажите пакетный менеджер (`npm` для Docker) и выполните другие конфигурации под себя.

#### 2.3. Установка навыка для Google Таблиц

```bash
clawhub install steipete/gog
```

### 3. (Опционально) Прокси для логирования запросов

Если вы хотите сохранять все обращения к модели (например, для отладки или анализа), можно запустить прокси‑сервер, который логирует HTTP‑запросы и ответы.

```bash
cd proxy
poetry install
poetry run python main.py
```

После этого добавьте в глобальный `.env` переменную:

```
OPENAI_PROXY=http://docker.host.internal:12434
```

(Замените хост и порт при необходимости.)

### 4. Подключение таблицы Google Sheets

1. **Найдите ID таблицы**  
   Откройте таблицу в браузере и скопируйте идентификатор из URL:  
   `https://docs.google.com/spreadsheets/d/***1f1AHqUptcMfQ7v6p1wViuefDcTgkLP9IB-M5GgWdESc***/edit`  
   Запишите его в `.env` как `GOG_TABLE`.

2. **Авторизуйтесь в Google API**  
   Выполните в консоли:

   ```bash
   gog auth credentials ~/Downloads/client_secret_....json

   # Шаг 1: получить URL для авторизации
   gog auth add you@gmail.com --services sheets --remote --step 1

   # Шаг 2: вставить полный redirect URL из адресной строки браузера
   gog auth add you@gmail.com --services sheets --remote --step 2 --auth-url 'http://127.0.0.1:<port>/oauth2/callback?code=...&state=...'
   ```

   Последняя команда запросит пароль для шифрования файла с токенами. Запишите этот пароль в `.env` как `GOG_PASSWORD`, а email – как `GOG_USER`.

### 5. Настройка пороговых значений

В файле `CONFIG.md` отредактируйте значения:

```
ПОРОГ_ПАДЕНИЕ_ВЫРУЧКИ = 20          # % падения относительно среднего за 7 дней
ПОРОГ_КОНВЕРСИЯ_МИН = 5              # минимальная конверсия в %
ПОРОГ_ДОПРОДАЖИ_ПАДЕНИЕ = 5         # % падения суммы допродаж
ПОРОГ_ОТСТАВАНИЕ_ОТ_ПЛАНА = 15       # % отставания факта от плана
```

Все пороги используются командой `/check` и периодическими проверками.

### 6. Синхронизация настроек

Укажите в `.env` путь к рабочей директории OpenClaw (обычно `~/.openclaw`).  
**Внимание:** Следующий шаг **заменяет** файлы в `~/.openclaw/agents/main/` и **модифицирует** конфигурационный файл `openclaw.json`, а также добавляет переменные окружения из `.env` в окружение бота.

Выполните:

```bash
./sync.sh
```

Скрипт скопирует файлы `AGENTS.md`, `CONFIG.md`, `TOOLS.md`, `CRON.md` (если есть) и другие необходимые ресурсы в рабочее пространство OpenClaw, а также добавит команды `/summary` и `/check` в `openclaw.json`.

### 7. Настройка периодических проверок (опционально)

Для автоматического запуска `/check` по расписанию откройте интерфейс OpenClaw, создайте новую cron‑задачу с нужным интервалом (например, каждый день в 9:00) и в качестве инструкции укажите `/check`.  
Чтобы уведомления приходили в Telegram, добавьте в задачу идентификатор нужного диалога (группы или личного чата).

## 📝 Логирование

Агент записывает каждый свой шаг в файлы `memory/YYYY-MM-DD.md`.  
Формат записи содержит время, запрос пользователя, контекст, действие, результат и ссылки на созданные скрипты.  
Это позволяет в любой момент спросить: «расскажи анализ по логам за последние 7 дней» – и получить сводку по обращениям.

Дополнительно, если запущен прокси (см. раздел 3), все HTTP‑запросы к модели сохраняются в `logs/` для глубокого анализа.

## 🤖 Бот

Бот работает через стандартный Telegram‑канал OpenClaw и поддерживает следующие команды (доступны в меню команд):

- `/summary` – сформировать сводку по данным, которые не были охвачены предыдущим отчётом (автоматически обновляет ячейку `Z1` в таблице).  
- `/check` – выполнить все проверки из `CHECKS` с актуальными порогами, сообщить о критических отклонениях.  

Любой естественный запрос (например, «какая конверсия за последние 7 дней?») также обрабатывается агентом.

## ⚠️ Ограничения

- Требуется мощная языковая модель с поддержкой **вызова инструментов** (function calling) и хорошими аналитическими способностями.
- Протестирована совместимость с:
  - ✅ **deepseek-reasoner** (облачный)
  - ✅ **Nemotron 3 Super** (облачный)
  - ❌ **qwen2.5:7b-instruct-q4_K_M**
  - ❌ **nemotron-3-nano:4b**
  - ❌ **granite4:3b**
  - ❌ **cogito:8b**
  - ❌ **deepseek-r1:8b** (не работает, так как OpenClaw не умеет вызывать с этой моделью инструменты)

- Прокси для логирования работает только с моделями, совместимыми с OpenAI API.
- Все расчёты выполняются скриптами на Python – необходима установка Python 3.8+ в окружении.

## 📁 Структура репозитория

- `AGENTS.md` – основная инструкция для ИИ‑агента (поведение, команды, логирование).  
- `CONFIG.md` – настраиваемые пороги и параметры.  
- `TOOLS.md` – описание доступных инструментов (gog, скрипты, память).  
- `CRON.md` – шаблон для периодических задач (при необходимости).  
- `sync.sh` – скрипт синхронизации конфигурации с рабочей директорией OpenClaw.  
- `proxy/` – прокси‑сервер для логирования запросов к моделям.  
- `.env.example` – пример файла с переменными окружения.  
- `Dockerfile.local`, `docker-compose.override.yml` – вспомогательные файлы для Docker‑установки.
